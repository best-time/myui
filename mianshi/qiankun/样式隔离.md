## strictStyleIsolation：ShadowDOM 严格样式隔离（彻底隔离）

#shadow-root

    import { registerMicroApps, start } from 'qiankun';
    registerMicroApps([/* 子应用列表 */], {
        sandbox: {
            strictStyleIsolation: true // 开启 ShadowDOM 隔离
        }
    });
    start();    

实现原理

    给每个子应用容器创建独立 shadowRoot，将子应用完整 DOM、样式全部封装进 ShadowDOM 内部。
    
    Web 原生特性：Shadow 内部 CSS 完全不会穿透到外部文档，外部全局样式也不会渗透进子应用，是真正物理级隔离。


优点
- 隔离强度最高，主子、子子之间完全无样式污染；
- 不会破坏 CSS 语法（无需动态改写样式代码）；
- @keyframes、全局重置样式、第三方 UI 库基础样式天然隔离。

致命缺点（高频踩坑）
- 弹窗 / Modal/Drawer/Popover 丢失样式 / 白屏
- 主流 UI 库（Element Plus、Ant Design）弹窗默认挂载到 document.body，而 body 不在 shadowRoot 内部，弹窗脱离隔离环境，样式全部失效；
- 主应用无法通过全局样式覆盖子应用；
- 部分 CSS 特性、字体、滚动、媒体查询存在兼容性问题；
- 子应用需要额外改造 UI 弹窗挂载节点，指向 shadow 内部容器。

    
    <!-- Vue3 Element Plus 示例 -->
    <el-config-provider append-to="#shadow-root-id"></el-config-provider>


### experimentalStyleIsolation：实验性 Scoped CSS 命名空间隔离（业务最常用）
    
    start({
        sandbox: {
            experimentalStyleIsolation: true // scoped 式前缀隔离
        }
    });

实现原理

    给子应用根容器自动添加唯一属性：data-qiankun="子应用名称"；
    运行时解析子应用所有 <style>、css 文件，给每一条 CSS 选择器自动追加容器属性前缀，类似 Vue 的 scoped 原理。

转换示例
    
    /* 子应用原始代码 */
    .btn { color: #fff; }
    body { margin: 0; }
    
    /* qiankun 自动转换后 */
    div[data-qiankun="app-user"] .btn { color: #fff; }
    div[data-qiankun="app-user"] body { margin: 0; }
优点
- DOM 仍挂载在全局 body，弹窗、抽屉、下拉框完全正常，无需改造子应用 UI；
- 兼容性远好于 ShadowDOM，几乎不需要业务改造；
- 主应用可通过更高权重选择器覆盖子应用样式。

缺点与踩坑
- 性能损耗：运行时解析、改写全部 CSS 文本，大型样式文件加载变慢；
- 全局选择器缺陷：html/body/:root/* 加前缀后失效，子应用全局重置样式会丢失；
- @keyframes 动画名称不会自动隔离，多子应用同名动画会冲突；
- CSS-in-JS 库（Ant Design 5.x）使用 :where() 低优先级选择器，容易被主应用样式覆盖；
- 动态插入的内联样式、JS 动态生成样式无法被捕获隔离。


### 补充业务层辅助隔离方案（搭配 qiankun 内置方案使用）
- CSS Module / Vue scoped / React styled-components
子应用自身组件级隔离，从根源减少全局样式冲突，推荐所有子应用强制开启；
- 统一命名空间前缀
所有子应用根容器添加唯一 class：app-user-xxx，手写 CSS 全部限定在此前缀下；
- 动态样式手动销毁
JS 动态创建的 style 标签，在子应用 unmount 生命周期手动移除；
- 避免全局选择器
禁止子应用使用 *、body、html 无差别全局样式覆盖。

### 生产环境推荐
- 后台管理系统、大量弹窗 / 表格场景：优先 experimentalStyleIsolation；
- 多实例、对外独立子应用、无复杂弹窗：优先 strictStyleIsolation（ShadowDOM）；
- 无论使用哪种内置隔离，子应用组件必须开启 scoped / CSS Module，双层防护；
- 不推荐仅使用默认模式，线上多子应用场景极易出现样式错乱。
